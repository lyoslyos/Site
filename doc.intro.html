Introduction
Installation
Après avoir téléchargé DrakkarD Framework vous pourrez commencer votre projet immédiatement.
Votre code sera à réaliser dans le dossier project de votre arborescence DrakkarD Framework, ce dossier project contient différents dossiers nécessaires au fonctionnement du framework: config, controllers, models et templates.
Dans le dossier web de l'arborescence du framework se trouve un fichier index.php qui est la base de départ de tout projet, vous n'aurez pas à y toucher. C'est par contre dans ce dossier que se trouvent tous les assets de votre projet: les images, les scripts, les feuilles de style, etc.
Autoload
Pour plus de facilité, Drakkard Framework intègre un autoloader de classes, ainsi plus besoin de devoir inclure chacun de vos fichiers de classe, à chaque appel de classe DrakkarD Framework recherche si cette classe existe et dans ce cas l'inclue.
Le dossier project contient un sous-dossier config, dans ce dossier doit se trouver un fichier classes.yml . C'est dans ce fichier que DrakkarD Framework va chercher le lien entre une classe et son fichier. A chaque fois que vous allez créer une classe dans votre projet, vous devrez absolument rajouter une ligne dans ce fichier classes.yml pour indiquer à l'autoloader où se trouve votre nouvelle classe:

Nom_de_la_classe : chemin/vers/le/fichier/de/la/classe.php

Le chemin vers le fichier se fait à partir du dossier project, voici un exemple pour une classe TestController dans project/controllers/TestController.php :

TestController : controllers/testController.php
Routeur
DrakkarD Framework vous propose un système de routing simple, qui avec un format d'url vous redirige vers une action spécifique.
Vous avez un premier fichier core/config/routing.yml qui redirige un format d'url vers un autre fichier de routing qui contient les controllers et les actions cibles.

nom_de_route :
    pattern : url_de_la_route
    ressource : chemin_vers_le_fichier_routing_du_projet

Il y est présent une route par défaut "drakkard" avec comme pattern par défaut "/", si vous souhaitez créer un autre chemin dans ce fichier il faut toujours que le tri des routes se fasse de façn décroissante. Si vous avez trois routes avec comem pattern "/", "/admin", "/admin/user", la première route à être présente dans le fichier sera la route "/admin/user", ensuite la route "/admin" et pour finir la route "/".

Ces routes dans le core du framework ammènent vers les routes de votre projet, ce sont ces routes qui indiquent quel format d'url ammène vers quelle action de quel controller.

nom_de_route :
    pattern : url_de_la_route
    controller : nom_du_controller
    action : nom_de_l'action

ATTENTION CONVENTION DE NOMMAGE: VOus devez nommer vos controllers sous ce format: "nomController" et vos action: "nomAction" MAIS dans vos routes vous devez juste indiquer les noms sans les suffixes "COntroller" et "Action".

Si vous avez besoin d'indiquer des paramètres d'url variables (par exemple un numéro de page) vous pouvez utiliser le format "{nom_de_la_variable}" dans le pattern de votre route, ainsi pour un numéro de page:

pattern : /page/{numero}
