Packages
Introduction
Le système de Package a été pensé pour vous permettre de façon très simple d'exécuter un flot de méthode en appelant simplement package avec l'instance en question ainsi que le nom du package a éffectué.

Pourquoi utiliser ce système ? tout d'abord pour la clarté du code ! Il est en effet désormais possible de ranger la plupart de ses actions dans de simple configurations. Ainsi votre action est représenté simplement.
La classe Package
La classe Package réponds au pattern des Singleton (ce qui signifie qu'il ne peut y avoir au plus qu'une seule instance de cette classe).

Pour y accéder il vous suffit de faire :

DrakkarD\Package\Package::getInstance();
Configuration d'un Package
La configuration des package se fait directement dans la classe que vous cibler. Vous créer un tableau qui contiendra en clé le nom de votre package et en valeur un tableau contenant le nom de toutes les méthodes a appelé.

Cet attribut doit être couplé d'une méthode publique getPackage() qui retourneras ce tableau.

Le nom des méthodes a appelé peut être couplé de différents arguments. L'ensemble des arguments devront être entourés de parenthèses. (comme lors d'un appel de fonction normal)
Les Arguments
Plusieurs types d'arguments peuvent être passés :

On peut passer une simple chaine de caractères entouré ou non de quote (') ou de double quote (").
function_to_call('chaine_de_caractères'); On peut aussi accéder a une variable globale. Pour ceci on va indiqué que l'on souhaite atteindre une variable grace au signe pourcentage (%)

ex : 'ActionName1' => 'functionToExecute(%variableName)'

On peut également accéder a une constante constante globale. Pour cela on va indiqué que l'on souhaite atteindre une constante grace au point d'exclamation (!)

ex : 'ActionName1' => 'functionToExecute(!CONSTANT_NAME)'

de la même manière on peut accéder aux :

instance et attribut d'instance

ex : 'ActionName1' => 'functionToExecute(%instanceName.attributName)'

instance variable protected ou private : Dans ce cas là on va tenter d'appeller la méthode getAttributName();

static attribut : 'ActionName1' => 'functionToExecute(%NamespaceName\InstanceName::%attributName)'

constantes de classe : 'ActionName1' => 'functionToExecute(%NamespaceName\InstanceName::!CONSTANT_NAME)'
